# Анализ багов и проблем в ZetGui репозитории

## 🚨 Критические проблемы

### 1. Небезопасность паролей (HIGH PRIORITY)
**Файл:** `src/main.ts:68`
```typescript
const password = await askQuestion(chalk.cyan('Password: ')); // TODO: скрыть пароль
```
**Проблема:** Пароли отображаются в открытом виде при вводе в консоли. Это критическая уязвимость безопасности.

**Решение:** Использовать библиотеку типа `read` или `readline-sync` с опцией скрытия ввода.

### 2. Проблемы с JSON парсингом (HIGH PRIORITY)
**Файл:** `src/core.ts:210-250`
**Проблема:** Множественные попытки исправления JSON с неэкранированными кавычками могут привести к некорректной работе AI сервиса.

**Проблемный код:**
```typescript
cleanedResponse = cleanedResponse.replace(
    /"code":\s*"([^"]*(?:\\.[^"]*)*?)"/g,
    (match: string, codeContent: string) => {
        const escapedCode = codeContent.replace(/(?<!\\)"/g, '\\"');
        return `"code": "${escapedCode}"`;
    }
);
```

### 3. Некорректное завершение программы
**Файл:** `src/main.ts:365`
```typescript
rl.close(); // TODO: улучшить способ остановки программы
```
**Проблема:** Резкое закрытие readline интерфейса может привести к проблемам с очисткой ресурсов.

## ⚠️ Потенциальные проблемы

### 4. Отсутствие сохранения файлов через Neutralino API


**Проблема:** Функция сохранения файлов не реализована полностью для desktop приложения.

### 5. Слабая обработка ошибок в Docker операциях
**Файл:** `backend/src/server.ts:100`
**Проблема:** Если Docker контейнер не найден, приложение может упасть с неинформативной ошибкой.

### 6. Проблемы с типизацией
**Проблема:** В `tsconfig.json` отключены многие строгие проверки TypeScript:
- `strictNullChecks` закомментирован
- `noImplicitReturns` отключен
- `noUncheckedIndexedAccess` отключен

Это может привести к runtime ошибкам с null/undefined значениями.

### 7. Версии зависимостей
**Файл:** `package.json`
**Проблема:** Использование плавающих версий (^) может привести к неожиданным breaking changes:
- `"axios": "^1.6.0"`
- `"chalk": "^4.1.2"`
- `"dockerode": "^4.0.0"`

### 8. Отсутствие проверки существования контейнера
**Файл:** `src/core.ts:334`
```typescript
if (!container) throw new Error(`Sandbox container '${SANDBOX_CONTAINER_NAME}' not found.`);
```
**Проблема:** Ошибка не очень информативная для пользователя и не предлагает решения.

### 9. Потенциальная утечка интервалов (Memory Leak)
**Файл:** `desktop/react-src/src/components/Footer/Footer.tsx:11-16`
```typescript
const interval = setInterval(() => {
  checkDockerStatus();
  fetchRemainingRequests();
}, 30000);

return () => clearInterval(interval);
```
**Проблема:** Хотя компонент правильно очищает интервал в useEffect cleanup, асинхронные операции внутри setInterval могут вызвать состояние гонки при размонтировании компонента.

### 10. Потенциальная Path Traversal уязвимость
**Файл:** `src/main.ts:278-280`
```typescript
const absPath = path.isAbsolute(targetFile)
    ? targetFile
    : path.join(process.cwd(), 'sandbox', targetFile);
```
**Проблема:** Если `targetFile` содержит `../`, злоумышленник может записывать файлы за пределами sandbox директории.
